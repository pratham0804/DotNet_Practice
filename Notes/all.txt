4. Small Architectural Improvement (Important)

Instead of:

Console.WriteLine("Course does not exist");

Services usually do not print to console.

Why?

Because services are used by:

Controllers

Background jobs

APIs

Other services

Instead, service should either:

return null

return bool

throw exception

Example better design:

if(course == null)
{
    return false;
}

Controller decides what message to show.

But for learning purposes, your current approach is fine.









5. What MapControllers() Actually Does

When ASP.NET starts, MapControllers() tells the routing system:

Scan the application
Find all controller classes
Register their routes
Connect them to the request pipeline

So ASP.NET looks for classes that:

inherit ControllerBase

Example:

public class StudentsController : ControllerBase

Then it reads the route attributes inside them.

Example:

[Route("api/students")]
[HttpGet]

and registers endpoints like:

GET /api/students
GET /api/students/{id}
POST /api/students
3. How It Fits in the Request Pipeline

Your pipeline roughly looks like:

Request
   ↓
Middleware
   ↓
Routing
   ↓
MapControllers()
   ↓
Controller Action

MapControllers() connects routing → controller methods.








6. ### ASP.NET Core Controller Return Types (Quick Notes)

**1. `IActionResult`**

* Used when the endpoint returns **only HTTP status responses**, not specific data types.
* Allows returning responses like:

  * `Ok()`
  * `NotFound()`
  * `BadRequest()`
  * `NoContent()`

Example:

```csharp
public IActionResult DeleteStudent(int id)
{
    var student = _studentService.GetStudentById(id);

    if(student == null)
        return NotFound();

    _studentService.DeleteStudent(id);

    return Ok();
}
```

---

**2. `ActionResult<T>`**

* Used when the endpoint returns **data + HTTP responses**.
* `T` represents the **data type returned when successful**.

Example:

```csharp
public ActionResult<Student> GetStudentById(int id)
{
    var student = _studentService.GetStudentById(id);

    if(student == null)
        return NotFound();

    return Ok(student);
}
```

---

**3. Returning Collections**

When returning multiple objects:

```csharp
public ActionResult<List<Student>> GetAllStudents()
{
    var students = _studentService.GetAllStudents();
    return Ok(students);
}
```

Better production pattern:

```csharp
public ActionResult<IEnumerable<Student>> GetAllStudents()
```

---

**4. Rule of Thumb**

| Scenario             | Return Type                    |
| -------------------- | ------------------------------ |
| Returning data       | `ActionResult<T>`              |
| Returning collection | `ActionResult<IEnumerable<T>>` |
| Only status codes    | `IActionResult`                |

---

**5. Helper Methods from `ControllerBase`**

Controllers inherit from `ControllerBase`, which provides:

* `Ok()` → **200**
* `NotFound()` → **404**
* `BadRequest()` → **400**
* `Created()` → **201**

These automatically create proper **HTTP responses**.
